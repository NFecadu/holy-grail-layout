This header has position: static (the default). Because static elements follow normal document flow, no top/right/bottom/left offsets apply here. It is 90% wide so it doesn’t stretch to the very edges, and I chose background-color: #20B2AA (LightSeaGreen) with a border: 3px solid #006400 (DarkGreen) from the Adobe Wheel to give a professional, harmonious look. This box sits at the top in the natural order.

This nav bar uses position: fixed, which removes it from normal document flow and pins it relative to the viewport. By setting top: 170px and left: 35px, it ends up exactly 170 pixels down from the top edge of the browser and 35 pixels from the left edge. Because it’s fixed, scrolling does not move it. I selected background-color: #9400D3 (DarkViolet) so it stands out clearly against the header’s LightSeaGreen and the white page background.

The .articles container uses position: relative. That means it stays in the normal flow but also serves as a containing block for any children that use position: absolute. Here, its own left: 30px moves it 30 pixels to the right from where it would normally sit (static position), and top: 45px moves it 45 pixels down. Because it is relative, those offsets shift this container without altering document flow, so other siblings (aside, footer) treat it as though it still occupies its original place.

This first <article> is also position: relative. That keeps it in normal flow inside its parent .articles. By setting top: 15px, the browser moves this box exactly 15 pixels down from where it would have appeared in the container. Because it is relative, the next <article> sits below, and no content overlaps. I chose background-color: #BA55D3 (MediumOrchid) with a border: 5px solid #808080 (Gray) to make it easy to identify in the layout.

This second <article> also uses position: relative. Its top: 15px shifts it 15 pixels down from its own default location inside the container—just like the first article—so both articles appear staggered by the same vertical amount. Matching colours (MediumOrchid and Gray border) ensure a cohesive, professional appearance.

The <aside> uses position: absolute, which removes it from the normal flow and positions it relative to the nearest ancestor that has a position other than static. In this code, no parent has an explicit position, so its containing block is the <body>. With left: 50%, its left edge sits halfway across the body’s width; and with top: 235px, it is 235 pixels down from the top of the body. Setting width: 40% means that by combining left:50% and width:40%, this sidebar consistently occupies the right column for a Holy Grail layout. I used background-color: #BDB76B (DarkKhaki) to contrast against the article’s MediumOrchid.

The <footer> is pinned with position: fixed and bottom: 0px. This makes it always flush to the bottom edge of the viewport, regardless of scrolling or content height. Its width: 100% ensures it spans the entire browser width, and at height: 50px plus padding, it perfectly fits in that bottom area. I picked background-color: #191970 (MidnightBlue) so it stands out as the final, distinct section.
