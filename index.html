This header has position: static (the default). Because static elements follow normal document flow, no top/right/bottom/left offsets apply. It is 90% wide (centered by margin: 25px auto) so it doesn’t stretch fully edge‐to‐edge. I chose background-color: #20B2AA (LightSeaGreen) with border: 3px solid #006400 (DarkGreen) from the Adobe Wheel to give a professional, harmonious look. This block simply sits at the top in normal order.

The nav bar uses position: fixed, which removes it from the normal flow and pins it relative to the viewport. By setting top: 135px and left: 5%, it ends up exactly 135 pixels down from the top edge (25px header margin + 100px header height + 10px header padding = 135px) and 5% in from the left edge. Because it is fixed, scrolling does not move it. I selected background-color: #9400D3 (DarkViolet) so it stands out clearly against the LightSeaGreen header.

The .articles container uses position: relative. That means it remains in the normal flow but becomes the reference for any absolutely positioned children (there are none here). Instead of using top/left inside .articles, I used margin: 20px 0 0 5% to push this entire block 20px down from the nav + 5% from the left edge. Because it’s relative, if we later choose to absolutely position something inside this container, the offsets would be computed from this container’s top-left corner.

This first <article> uses position: relative. That keeps it in the normal flow, but allows us to nudge it with top: 15px. In other words, the browser first lays it out where it would normally sit inside .articles, then “slides it” 15px down from that spot. Because it remains in flow, the second article just below also moves down accordingly. I chose background-color: #BA55D3 (MediumOrchid) with a border: 5px solid #808080 (Gray) so it’s easy to spot in the left column.

This second <article> also uses position: relative. Its top: 15px shifts it 15px down from where it would normally appear inside its parent .articles. By matching the same vertical offset as the first article, both boxes appear aligned but each is pushed downward the same amount. The repeated MediumOrchid and Gray border keep a cohesive look.

The <aside> uses position: absolute, which removes it from the normal document flow and places it relative to the nearest positioned ancestor. Because no parent

has a position property, it falls back to the <body> as its containing block. Setting top: 135px makes it 135px from the top of the page (the same vertical offset as the nav), and right: 5% places its right edge 5% in from the viewport’s right edge. Because its width is 40%, this combination ensures it occupies the correct right‐hand column for our Holy Grail layout. I used background-color: #BDB76B (DarkKhaki) so it contrasts nicely with the left articles.

The <footer> is pinned with position: fixed and bottom: 0, which means it always stays flush to the bottom edge of the viewport regardless of scrolling. Its width: 100% spans the entire browser width; height: 50px plus padding fits neatly at the bottom. I chose background-color: #191970 (MidnightBlue) so it stands out as a distinct final section.
